   - `default-features = false` on sqlx → rustls, no
     native-tls.
2. Module layout — pattern after http_forward. Suggested:
   - `src/lib.rs` — crate-root rustdoc + module plumbing +
     public `SqlMysql` type + `impl Implementation for
     SqlMysql` + impl-api re-exports.
   - `src/config.rs` — `SqlMysqlConfig` with
     `connection_url`, `max_connections`,
     `default_timeout_ms`, `default_max_rows`;
     `deny_unknown_fields`; `prepare()` builds a `MySqlPool`.
     `connection_url` must start with `mysql://` (or
     `mariadb://`, which sqlx accepts as an alias); other
     schemes fail at config-ingest time.
   - `src/request.rs` — `SqlQueryRequest` with `sql`,
     `params`, optional `max_rows`, optional `timeout_ms`;
     `deny_unknown_fields`.
   - `src/response.rs` — `SqlQueryResponse` with `rows`,
     `row_count`, `columns`, `truncated`; `Column` with
     `name`, `sql_type`.
   - `src/execute.rs` — core execute logic: parameter
     binding, query execution with per-request timeout,
     row collection with `max_rows` truncation, column
     metadata extraction.
   - `src/types.rs` — SQL-to-JSON value conversion per
     doc 08 §"SQL-to-JSON type mapping". Specific MySQL
     notes:
     - **Unsigned integer types** (TINYINT UNSIGNED,
       SMALLINT UNSIGNED, MEDIUMINT UNSIGNED, INT UNSIGNED,
       BIGINT UNSIGNED): surface as JSON number when they
       fit in i64; BIGINT UNSIGNED values above i64 range
       (> 2^63 - 1) → `UpstreamError` per doc 08's
       overflow-returns-upstream_error rule. Document the
       quirk in rustdoc.
     - **DECIMAL / NUMERIC** → JSON string (avoid float
       precision loss).
     - **DATETIME / TIMESTAMP** → JSON string RFC 3339.
       MySQL TIMESTAMP is UTC-normalized on storage; emit
       as `...Z`-suffixed strings. MySQL DATETIME is
       zone-naive; emit without a zone suffix if that's
       what sqlx produces, and note in rustdoc that scripts
       treating those values as UTC is at script's risk.
     - **JSON** (MySQL 5.7+) → JSON value verbatim.
     - No ARRAY type in MySQL (Postgres-only). If some
       script sends MySQL an array param, surface as
       `InvalidRequest`.
   - `src/error.rs` — internal `Error` enum + `From<Error>
     for ImplementationError`.

### Decisions fixed upstream (do NOT deviate)

1. **`sqlx = "0.8"`**, `default-features = false` +
   `["mysql", "runtime-tokio-rustls", "json", "chrono",
   "uuid"]`. rustls, not native-tls.
2. **No connector-level retry.**
3. **`UpstreamError.status` for DB errors**: `500` sentinel;
   `body` carries the formatted sqlx error message.
4. **`max_rows` truncation via streaming**: use
   `sqlx::query(...).fetch(&pool)` + `StreamExt::take(max_rows
   + 1)`; truncate extra row, set `truncated: true`. Do NOT
   inject `LIMIT` into the script's SQL.
5. **`timeout_ms` enforcement**: `tokio::time::timeout(...)`
   wrapping query execution at effective timeout (request
   override or config default, whichever is smaller). On
   timeout → `UpstreamTimeout`.
6. **Clamping**: request values for `max_rows` / `timeout_ms`
   may override config defaults downward only; upward values
   are silently clamped (doc 08 treats config as a cap).
7. **Implementation name**: `Implementation::name()` returns
   **`"sql_mysql"`**.
8. **Integer range**: i64 for signed; BIGINT UNSIGNED above
   i64 range → `UpstreamError`. DECIMAL/NUMERIC → JSON
   string.

## Error mapping

| Doc 08 case              | Internal `Error` variant                | Wire `ImplementationError`       |
| ---                      | ---                                     | ---                              |
| `invalid_sql`            | `InvalidSql(detail)`                    | `InvalidRequest { detail }`      |
| `parameter_mismatch`     | `ParameterMismatch { expected, actual }`| `InvalidRequest { detail: "parameter count mismatch: expected N, got M" }` |
| `upstream_error`         | `UpstreamDbError(sqlx_error_string)`    | `UpstreamError { status: 500, body: <formatted> }` |
| `upstream_timeout`       | `UpstreamTimeout`                       | `UpstreamTimeout`                |
| `upstream_unreachable`   | `UpstreamUnreachable(detail)`           | `UpstreamUnreachable { detail }` |
| Integer range overflow   | `IntegerOverflow { column, value }`     | `UpstreamError { status: 500, body: <formatted> }` |
| Config deser fails       | `InvalidConfig(detail)`                 | `InvalidConfig { detail }`       |
| Request deser fails      | `InvalidRequest(detail)`                | `InvalidRequest { detail }`      |
| Internal bug             | `Internal(detail)`                      | `Internal { detail }`            |

MySQL `sqlx::Error` classification guide:
- `Error::Database(db_err)` with a SQLSTATE class
  `42xxx` (or MySQL code 1064 / 1054 / 1146 / 1149 — syntax,
  unknown column, unknown table, parser error) → `InvalidSql`.
- `Error::Database(db_err)` with any other SQLSTATE →
  `UpstreamDbError`.
- `Error::Io(_)` / `Error::PoolTimedOut` / `Error::Protocol(_)`
  → `UpstreamUnreachable`.
- `Error::RowNotFound` / decoding errors → `Internal`.

Flag any ambiguous case under Residual risks.
