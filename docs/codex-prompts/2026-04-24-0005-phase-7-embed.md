## Motivation

Phase 7 Tier 1 connector implementation: `embed`, the local
in-process embedding connector. No HTTP, no HF runtime
download, no API keys. The deployment's connector-service
binary `include_bytes!`s an ONNX embedding model + its
tokenizer files at build time; `Embed::new_from_bytes(...)`
eagerly loads them; `execute(...)` runs inference locally
via `fastembed`. Dispatched in parallel with its sibling
`2026-04-24-0006-phase-7-vector-search.md`; the two crates
are independent and will land in separate submodule commits.

Non-crypto task: no Gate 1/2, no key material.

     ["derive"] }`, `serde_json = "1"`, `thiserror = "2"`.
   - Dev-deps: `tokio = { version = "1", features = ["rt",
     "macros", "time", "test-util"] }`.
   - **No `reqwest`**, no TLS stack — this crate has no
     network.
2. Module layout per the impl spec's `embed` §"Module layout":
   - `src/lib.rs` — crate rustdoc + public `Embed` type +
     `Embed::new_from_bytes(...)` constructor + trait impl +
     impl-api re-exports + re-export of fastembed's
     `TokenizerFiles`.
   - `src/config.rs` — `EmbedConfig` (`model_id`,
     `max_batch_size`, `timeout_ms`); `deny_unknown_fields`;
     default-impl helpers (`default_max_batch_size() -> 32`,
     `default_timeout_ms() -> 10_000`).
   - `src/request.rs` — `EmbedRequest` with `texts:
     Vec<String>` + `deny_unknown_fields`.
   - `src/response.rs` — `EmbedResponse` with `embeddings:
     Vec<Vec<f32>>`, `model: String`, `dimensions: usize`.
   - `src/model.rs` — thin wrapper around
     `fastembed::TextEmbedding::try_new_from_user_defined(...)`.
     Holds the loaded `TextEmbedding` instance +
     `model_id`, `dimensions`, `max_seq_length`. Exposes
     one `embed(&self, texts: &[String]) -> Result<Vec<Vec<f32>>,
     Error>` method.
   - `src/error.rs` — internal `Error` enum + `From<Error>
     for ImplementationError`.
3. Public surface (per impl spec):

   ```rust
   pub struct Embed { /* private */ }

   impl Embed {
       pub fn new_from_bytes(
           model_id: impl Into<String>,
           onnx_bytes: Vec<u8>,
           tokenizer_files: TokenizerFiles,
           dimensions: usize,
           max_seq_length: usize,
       ) -> Result<Self, ImplementationError>;

       pub fn model_id(&self) -> &str;
       pub fn dimensions(&self) -> usize;
   }

   #[async_trait]
   impl Implementation for Embed { ... }

   // Re-exports
   pub use fastembed::TokenizerFiles;
   pub use philharmonic_connector_impl_api::{
       Implementation, ImplementationError, ConnectorCallContext,
       JsonValue, async_trait,
   };
   ```

   `Embed::new_from_bytes` eagerly loads the model (fastembed
   validates the ONNX + tokenizer bundle during
   `try_new_from_user_defined`). Failure maps to
   `ImplementationError::InvalidConfig { detail }` (a
   deployment-supplied malformed model is a config issue
   from the impl's perspective, even though it manifests at
   construction rather than at execute).

4. `execute(config, request, ctx)` flow:
   - Deserialize `config` → `EmbedConfig`. `InvalidConfig`
     on failure.
   - Validate `config.model_id == self.model_id`. If
     mismatch → `InvalidConfig { detail: "config model_id
     'X' does not match loaded model 'Y'" }`.
   - Deserialize `request` → `EmbedRequest`. `InvalidRequest`
     on failure.
   - Validate `request.texts.len() >= 1` and `<=
     config.max_batch_size`. Violations → `InvalidRequest`.
   - Spawn inference on `tokio::task::spawn_blocking(...)`
     (fastembed's inference is CPU-bound and blocking).
     Wrap the spawn in `tokio::time::timeout(config.timeout_ms,
     ...)` — on timeout, return `UpstreamTimeout`. On
     spawn_blocking join error → `Internal`. On fastembed
     error → `Internal { detail: <error> }`.
   - Serialize `EmbedResponse { embeddings, model:
     self.model_id.clone(), dimensions: self.dimensions }`
     to JSON. Return.

5. Unit tests colocated with each module:
   - `config::tests` — deny-unknown-fields, defaults for
     `max_batch_size` / `timeout_ms`, model_id required.
   - `request::tests` — deserialize, reject non-string
     texts, reject empty array, reject missing field.
   - `response::tests` — round-trip serialize/deserialize.
   - `error::tests` — every internal variant maps to the
     right wire error.
   - No `model::tests` that actually load an ONNX —
     inference is exercised via the env-var-gated
     integration tests (below).

6. Integration tests under `tests/`:
   - **All `#[ignore]`-gated by default** and opt-in via
     env vars:
     - `EMBED_TEST_ONNX_PATH=/path/to/model.onnx`
     - `EMBED_TEST_TOKENIZER_DIR=/path/to/tokenizer-files/`
       (contains `tokenizer.json`, `tokenizer_config.json`,
       `config.json`, `special_tokens_map.json`)
     - `EMBED_TEST_MODEL_ID=<string>`
     - `EMBED_TEST_DIMENSIONS=<int>`
     - `EMBED_TEST_MAX_SEQ_LENGTH=<int>`
   - Tests (all `#[ignore]`-d):
     - `inference_produces_correct_shape.rs` — single text
       in, one vector out; vector length matches
       `dimensions`; model-echo matches env `MODEL_ID`.
     - `batch_inference.rs` — batch of 3 texts → 3 vectors.
     - `deterministic_output.rs` — same input twice →
       identical output vectors (floating-point equality
       OK for deterministic ONNX inference at the same
       precision).
     - `semantic_similarity.rs` — cosine(embed("hello"),
       embed("hi")) > cosine(embed("hello"),
       embed("goodbye")). Sanity check; doesn't pin exact
       numeric values.
     - `batch_size_enforcement.rs` — `max_batch_size` cap
       rejects overlarge requests with `InvalidRequest`.
   - `tests/README.md` documents the env-var gating +
     how a developer materializes a model locally (sketch:
     pre-running the `hf-fetch-embed-model` xtask with a
     specific model, or manually downloading).

7. `CHANGELOG.md` — `[0.1.0] - 2026-04-24` entry; drop any

8. Crate-root rustdoc on `src/lib.rs` — density matching

### Decisions fixed upstream (do NOT deviate)

From the impl spec's §"Decisions (resolved 2026-04-24)" +
§"Residual decisions":

1. **fastembed v4** with `TextEmbedding::try_new_from_user_defined(...)`
   exclusively. Verify multilingual-model support works
   (the spec names `paraphrase-multilingual-MiniLM-L12-v2`
   as the canonical target; others are acceptable if
   fastembed supports them). If fastembed has regressed
   on multilingual — flag and stop rather than
   silently switching libraries.
2. **No network, no HF download, no disk I/O in the crate.**
   All weights flow through `new_from_bytes`.
3. **Eager model load at construction time.** A malformed
   bundle fails `new_from_bytes`, not `execute`.
4. **One model per `Embed`.** No internal model registry
   (Qe5→A).
5. **Env-var-gated integration tests; no ONNX in repo**
   (Qe6→A).
6. **`model_id` echo validation at execute time.** Config's
   `model_id` must match the loaded model's identifier;
   mismatch is `InvalidConfig`.
7. **Implementation name**: `Implementation::name()` returns
   **`"embed"`**.
8. **Config's `max_batch_size`** default: 32.
   `timeout_ms` default: 10_000.

  `unimplemented!` on reachable paths, no unbounded
  indexing, no unchecked integer arithmetic, no lossy

  because fastembed has both a path-based API
