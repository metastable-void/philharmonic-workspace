## Scope

### In scope

#### 1. Config struct (`philharmonic/src/bin/philharmonic_api/config.rs`)

```rust
#[derive(Debug, serde::Deserialize)]
#[serde(default)]
pub struct ApiConfig {
    pub bind: SocketAddr,          // default 127.0.0.1:3000

    // Database
    pub database_url: String,      // MySQL connection string

    // API token signing
    pub signing_key_path: Option<PathBuf>,  // Ed25519 seed (32 bytes)
    pub signing_key_kid: Option<String>,
    pub issuer: String,            // default "philharmonic"

    // API token verification
    pub verifying_keys: Vec<VerifyingKeyConfig>,

    // SCK (substrate confidentiality key)
    pub sck_path: Option<PathBuf>,
    pub sck_key_version: i64,      // default 1

    // Connector router (embedded)
    pub connector_dispatch: HashMap<String, UpstreamConfig>,
    pub connector_domain_suffix: String,

    // Rate limiting
    pub rate_limit: Option<RateLimitOverrides>,

    // TLS
    #[cfg(feature = "https")]
    pub tls: Option<TlsFileConfig>,
}

#[derive(Debug, serde::Deserialize)]
pub struct VerifyingKeyConfig {
    pub kid: String,
    pub public_key_path: PathBuf,
    pub issuer: String,
}

#[derive(Debug, serde::Deserialize)]
pub struct RateLimitOverrides {
    // Optional per-family overrides; the library crate has
    // sensible defaults via RateLimitConfig::default()
}
```

Exact shapes depend on what the library APIs accept ‚Äî read
them and match. The above is a guideline.

#### 2. Request scope resolver

The API library requires a `RequestScopeResolver` impl.
For the bin target, create a simple implementation:

- Reads a `X-Tenant-Id` header ‚Üí `RequestScope::Tenant(uuid)`
- If no header ‚Üí `RequestScope::Operator`

This is a placeholder for real deployment-specific resolution.
Name it `HeaderBasedScopeResolver`.

#### 3. Main binary (`philharmonic/src/bin/philharmonic_api/main.rs`)

Follow the mechanics-worker / connector pattern:

1. Clap CLI with `BaseCommand`.
2. TOML config loading with graceful fallback.
3. Connect to MySQL via `sqlx::MySqlPool::connect(&database_url)`.
4. Build `SqlStore<SinglePool>` from the pool.
5. Load signing key from file (32-byte Ed25519 seed) ‚Üí
   `ApiSigningKey::from_seed(seed, kid)`.
6. Build `ApiVerifyingKeyRegistry` from `verifying_keys`
   config entries (read public key files, register each
   with kid + issuer).
7. Optionally load SCK from file ‚Üí `Sck::from_bytes(bytes)`.
8. Build connector router: `DispatchConfig` from
   `connector_dispatch` ‚Üí `RouterState` ‚Üí router.
9. Build `PhilharmonicApiBuilder`:
   - `.request_scope_resolver(Arc::new(HeaderBasedScopeResolver))`
   - `.store(Arc::new(store))`
   - `.api_verifying_key_registry(registry)`
   - `.api_signing_key(signing_key)`
   - `.issuer(config.issuer)`
   - `.step_executor(Arc::new(StubExecutor))`
   - `.config_lowerer(Arc::new(StubLowerer))`
   - `.sck(sck)` (if loaded)
   - `.key_version(config.sck_key_version)`
   - `.extra_routes(connector_router)` (nest at `/connector/`)
   - `.build()?`
10. Start HTTP(S) server with `api.into_make_service()`.
11. SIGHUP reload: re-read config, rebuild verifying key
    registry + connector dispatch. Database pool and signing
    key are long-lived (not reloaded). Log what changed.

### 3. `main.rs`

Follow the mechanics-worker / connector pattern:

a. Clap CLI with `Option<BaseCommand>`, defaulting to `Serve`.
b. `resolve_config_paths("api", &args)`, load config.
c. Connect to MySQL:
   ```rust
   let pool = sqlx::MySqlPool::connect(&config.database_url)
       .await
       .map_err(|e| format!("database connection failed: {e}"))?;
   let store = SqlStore::new(SinglePool::new(pool));
   ```
d. Load signing key (if configured):
   read 32-byte seed file ‚Üí `ApiSigningKey::from_seed(...)`.
   Read the `ApiSigningKey` constructor to get the exact
   signature.
e. Build `ApiVerifyingKeyRegistry` from config entries.
   Read `ApiVerifyingKeyEntry` and `ApiVerifyingKeyRegistry`
   APIs to match exactly.
f. Load SCK (if configured): read file ‚Üí `Sck::from_bytes(...)`.
g. Build connector router (if dispatch table non-empty):
   `DispatchConfig` ‚Üí `RouterState` ‚Üí `router()`. Nest at
   a path like `/connector/` via `extra_routes`.
h. Build `PhilharmonicApiBuilder`:
   - `.request_scope_resolver(Arc::new(HeaderBasedScopeResolver))`
   - `.store(Arc::new(store))`
   - `.api_verifying_key_registry(registry)`
   - `.api_signing_key(signing_key)` (if loaded)
   - `.issuer(config.issuer.clone())`
   - `.step_executor(Arc::new(StubExecutor))`
   - `.config_lowerer(Arc::new(StubLowerer))`
   - `.sck(sck)` (if loaded)
   - `.key_version(config.sck_key_version)`
   - `.extra_routes(connector_router)` (if built)
   - `.build()?`
i. Start HTTP(S) server using the router from
   `api.into_router()` with `axum::serve` or hyper.
j. SIGHUP reload loop: re-read config, rebuild verifying key
   registry + connector dispatch config. DB pool and signing
   key are long-lived. Log changes.
