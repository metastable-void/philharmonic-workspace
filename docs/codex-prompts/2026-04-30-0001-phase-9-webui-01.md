## API endpoints the WebUI must call

### Meta
- `GET /v1/_meta/version`
- `GET /v1/_meta/health`

### Workflow templates
- `POST /v1/workflows/templates` — create
- `GET /v1/workflows/templates` — list (paginated)
- `GET /v1/workflows/templates/{id}` — read
- `PATCH /v1/workflows/templates/{id}` — update
- `POST /v1/workflows/templates/{id}/retire` — retire

### Workflow instances
- `POST /v1/workflows/instances` — create
- `GET /v1/workflows/instances` — list (paginated)
- `GET /v1/workflows/instances/{id}` — read
- `GET /v1/workflows/instances/{id}/history` — history
- `GET /v1/workflows/instances/{id}/steps` — steps
- `POST /v1/workflows/instances/{id}/execute` — execute step
- `POST /v1/workflows/instances/{id}/complete` — complete
- `POST /v1/workflows/instances/{id}/cancel` — cancel

### Audit
- `GET /v1/audit` — list (paginated, filterable)

### Tenant
- `GET /v1/tenant` — read settings
- `PATCH /v1/tenant` — update settings

Auth: All requests carry `Authorization: Bearer pht_<token>`.

## Scope

### In scope

Write the React/Redux/TypeScript source files under
`philharmonic/webui/src/`. The project scaffold (package.json,
tsconfig, webpack.config, index.html) already exists.

#### Pages (React Router)

1. **Login** (`/login`) — text input for pasting a `pht_`
   long-lived API token. Store in Redux (and sessionStorage).
   Redirect to `/` on success. No username/password — just
   the token string.

2. **Dashboard** (`/`) — show API version + health status.
   Redirect to `/login` if no token.

3. **Workflow Templates** (`/templates`) — list with pagination.
   Click to view detail. "Create" button opens a form.

4. **Template Detail** (`/templates/:id`) — show template
   fields. "Update" and "Retire" buttons.

5. **Workflow Instances** (`/instances`) — list with pagination.
   "Create" button (select a template, submit).

6. **Instance Detail** (`/instances/:id`) — show instance
   state, history, steps. "Execute Step", "Complete",
   "Cancel" action buttons.

7. **Audit Log** (`/audit`) — paginated list with date
   filter controls.

8. **Tenant Settings** (`/settings`) — read/update tenant
   display name.

#### Redux store (Redux Toolkit)

- `auth` slice: token string, isAuthenticated flag.
- `api` slice (or RTK Query): API client with Bearer token
  injection. Base URL is `/v1/` (same origin, proxied by
  webpack-dev-server in dev, served by the API bin in
  production).

#### Shared components

- `<Layout>` — nav sidebar + main content area.
- `<ProtectedRoute>` — redirect to `/login` if not authed.
- `<Pagination>` — cursor-based pagination controls
  (the API uses opaque cursors, not page numbers).
- `<JsonViewer>` — display JSON config/request/response
  objects in a readable format.

#### Styles

Minimal, clean CSS. No CSS framework (keep the bundle small).
Use `app.css` for global styles. Component-level styles via
CSS modules or just class names in `app.css`.

### Auth flow

Login page: user pastes a `pht_...` token string. Store in
Redux `authSlice` and `sessionStorage`. All API calls read the
token from the store and set `Authorization: Bearer <token>`.
If a 401 is returned, clear the token and redirect to `/login`.

### API client (`api/client.ts`)

A thin `fetch` wrapper:
```typescript
async function apiCall<T>(
  path: string,
  options?: RequestInit
): Promise<T>
```
- Prepends `/v1/` to the path.
- Injects `Authorization: Bearer <token>` from Redux store.
- Sets `Content-Type: application/json` for POST/PATCH.
- Parses JSON response.
- On 401: clear auth, redirect to login.
- On other errors: throw with status + error body.

### Pages

1. **Login** — text input + "Login" button. Validates the
   token starts with `pht_` before accepting. Stores in
   Redux + sessionStorage. Redirects to `/`.

2. **Dashboard** — calls `GET _meta/version` and
   `GET _meta/health`. Displays version string and health
   status.

3. **Templates** — `GET workflows/templates` with pagination.
   Table: id, display name, status, created. "Create" button
   → form with name + abstract config (JSON textarea). On
   submit: `POST workflows/templates`.

4. **Template Detail** — `GET workflows/templates/:id`.
   Shows all fields. "Update" button → form (PATCH).
   "Retire" button → `POST .../retire`.

5. **Instances** — `GET workflows/instances` with pagination.
   Table: id, template, state, created. "Create" button →
   pick template + submit.

6. **Instance Detail** — `GET workflows/instances/:id`.
   Shows state, template ref. Tabs: History
   (`GET .../history`), Steps (`GET .../steps`). Action
   buttons: Execute (`POST .../execute`), Complete
   (`POST .../complete`), Cancel (`POST .../cancel`).

7. **Audit Log** — `GET audit` with pagination. Optional
   date range filter. Table: timestamp, event type,
   principal, details.

8. **Tenant Settings** — `GET tenant` to read. Form to
   update display name (`PATCH tenant`).
