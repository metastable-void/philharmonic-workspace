## Non-obvious design call

`InstanceStatus::can_transition_to` includes `Pending -> Failed`.

Reason: the execution-sequence spec requires script errors and malformed
results to transition the instance to `Failed`. A first step can fail
while the instance is still `Pending` (revision 0), and the engine writes
only one instance revision per step execution. Without `Pending -> Failed`,
first-step failures would produce an invalid transition and violate the
failure-handling requirement.
