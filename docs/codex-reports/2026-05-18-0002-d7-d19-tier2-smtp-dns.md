# D7 + D19 Tier-2 SMTP/DNS Connector Notes

**Date:** 2026-05-18
**Prompt:** docs/codex-prompts/2026-05-18-0002-d7-d19-tier2-connectors-smtp-dns-01.md

The SMTP connector uses `lettre 0.11` with
`default-features = false` and the feature set
`["aws-lc-rs", "builder", "smtp-transport", "tokio1",
"tokio1-rustls", "webpki-roots"]`. I deliberately avoided
`tokio1-rustls-tls` / `rustls-tls` because lettre 0.11.22 maps
`rustls-tls` to `ring`; the selected lower-level rustls feature
plus `aws-lc-rs` and `webpki-roots` gives the required provider
and trust-store shape.

SMTP tests cover the locked connection-mode matrix, port-25 and
credential validation, TLS strictness plan selection, and MIME
envelope normalization. I did not add a live SMTP mock-server
integration test in this dispatch; the reliable local surface is
the config-to-transport plan and MIME/request validation. The
actual submission path is still exercised by the type system and
focused crate checks, but not by a networked server fixture.

The DNS connector needed no `mechanics-dns` public-surface
changes. `mechanics-dns` already re-exported `RecordType`,
`ResponseCode`, `parse_record_type`, and `DnsRecord`, which were
enough for D19 without any direct `hickory-*` dependency.

DNS tests cover the policy gate before resolver I/O, including
allowlist, blocklist, both-list overlay-deny, RR type allowlists,
empty names, unknown RR types, timeout selection, timeout clamps,
and synthetic RCODE mapping for NXDOMAIN / SERVFAIL / NOTIMP /
REFUSED. I did not wire a controlled DNS server fixture; RCODE
mapping is unit-tested with synthetic `mechanics_dns::Error`
values to avoid making CI depend on local UDP/TCP listener
behavior.
