## Overall status

Philharmonic’s workspace and crate family are in a post-v1 state: the v1 implementation path is complete through Phase 9, the reference deployment is operational, and the verified end-to-end workflow/RAG stack is in place. Current active work is no longer on core platform bring-up, but on hardening and cleanup around the HTTP client/runtime path, while the main MVP-blocking item has been narrowed to scoping and building a separate production Chat UI for the customer-support use case.

## What is next

Per `ROADMAP.md`, the next dispatchable and MVP-blocking work is §3.M: scoping and then building a production Chat UI for the customer-support use case, in a separate codebase from the framework crate family (either as an in-tree top-level app/bin or as an external project consuming the published crates). That scoping still needs the codebase-shape decision, product requirements for the customer-support flow, and deployment/consumption choices. Tier-3 Anthropic and Gemini connectors remain explicitly deferred to post-MVP and are not queued for the current cycle.

## Notable concerns or blockers

- MVP deployment is still blocked on the production Chat UI; the framework itself is operational, but the required end-user-facing chat application has not yet been scoped and built.
- One deferred item remains unscoped in the roadmap: the crypto-review-aware `lowerer.rs` / `embed_job.rs` extraction slice, which awaits a Gate-1 proposal before sequencing.
